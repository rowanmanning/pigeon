Text-Level Semantics

The strong and b elements look similar but have different purposes: strong is used to indicate important text and b is used to offset text from its surroundings with no additional importance inferred. E.g. strong and b.

The em and i elements look similar but have different purposes: em is used to indicate emphasised text where the emphasis is important to the meaning of the text and i is used (odd, I know) to indicate that the text may be spoken with an inflection but bears no extra importance. E.g. em and i.

The sub and sup elements do just what they say on the tin: mark up subscript/superscript. E.g. sub and sup.

The abbr element is used to mark up abbreviations, and is expected to have a title attribute. E.g. abbr.

The del and ins elements are used to mark up text that has been removed from the document or inserted after it was published. E.g. deleted inserted. The s element is subtly different from del – it is used to indicate that information is no longer accurate or relevant. E.g. strike.

The cite element should wrap the content of an anchor which links to a cited source. E.g. see this article

The q element is used to mark up inline quotations. E.g. Hello World!.

The mark element is used to highlight text, often search terms. E.g. You searched for kittens.

The small element is used to represent small print and side comments. E.g. No kittens were harmed in the making of this website*

* Maybe

Unordered Lists

ul elements should be used when the order of the list items is not important or has no meaning, e.g. a shopping list. They look like this:

  • Meat
  • Vegetables
    • Potatoes
    • Carrots
    • Peas
  • Fruit
    • Apples
    • Bananas

Ordered Lists

ol elements should be used when the order of the list items is important, and the list items would make no sense when reordered, e.g. a recipe. They look like this:

  1. Open tin of beans
  2. Cook beans
    1. Empty beans into pan or microwave-safe container
    2. Put on hob or in microwave
    3. Cook for a few minutes
  3. Make toast
    1. Place bread in toaster and cook
    2. Butter toasted bread
  4. Put beans on toast

Description Lists

dl elements should be used when the list contains items with a clear title/description. E.g. a list of books/reviews. Because dt and dd elements do not share a common parent, it can be difficult to use description lists for complex data. In this case, using a ul is sometimes considered acceptable. Code

As well as the code element being usable inline, often it is used in conjunction with the pre element. This allows marking up of larger blocks of code: 

function sayHelloTo (name) {
    alert('Hello ' + name + '!');
}

sayHelloTo('World');
